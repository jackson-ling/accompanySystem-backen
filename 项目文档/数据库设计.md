# 陪护系统数据库设计

> 适用于 Vue3 前端 + Spring Boot 后端

---

## 一、项目概述

这是一个陪护/陪诊服务系统，提供以下核心功能：
- **用户端**：浏览服务、预约陪诊、订单管理、充值缴费、收藏、消息通讯
- **陪诊师端**：工作台统计、抢单大厅、订单管理、收入明细
- **AI功能**：智能问答服务

---

## 三、字段转换说明

### 1. collected 字段说明
`collected` 字段**不存储在数据库中**，而是在查询陪诊师列表时通过关联 `favorite` 表动态计算返回。

```sql
-- 查询示例：关联 favorite 表判断是否收藏
SELECT c.*,
       CASE WHEN f.id IS NOT NULL THEN true ELSE false END AS collected
FROM companion_profile c
LEFT JOIN favorite f ON f.item_id = c.id AND f.type = 'companion' AND f.user_id = ?
```

### 2. 订单ID格式转换
- **数据库存储**: BIGINT 自增ID
- **API返回**: 格式化为字符串 `"ORD" + 年月日 + 6位序号`（如 `ORD20240315001`）

后端在序列化响应时做转换：
```java
// OrderDTO.java
@JSONField serializeUsing = OrderIdSerializer.class)
private Long id;
```

### 3. 字段命名风格
- **数据库字段**: 下划线命名 (snake_case)
- **API响应**: 驼峰命名 (camelCase)

后端可使用 Jackson 的 `@JsonProperty` 或 MapStruct 自动转换。

## 四、表关系图

```
用户 (sys_user)
  ├── 1:N → 就诊人 (patient)
  ├── 1:N → 订单 (orders)
  ├── 1:N → 充值记录 (recharge_record)
  ├── 1:N → 消费记录 (transaction_record)
  ├── 1:N → 收藏 (favorite)
  ├── 1:N → 反馈 (feedback)
  ├── 1:N → 消息会话 (message_conversation)
  └── 1:N → AI聊天记录 (ai_chat_history)

陪诊师信息 (companion_profile) ←→ sys_user(通过user_id关联)
  ├── 1:N → 订单 (orders)
  └── 1:N → 收入记录 (companion_income)

服务分类 (service_category)
  └── 1:N → 服务项目 (service_item)

服务项目 (service_item)
  └── 1:N → 订单 (orders)

订单 (orders)
  └── 1:N → 评价 (order_review)

医院 (hospital)
  └── 1:N → 科室 (department)

消息会话 (message_conversation)
  └── 1:N → 聊天消息 (chat_message)
```

### SQL 示例

## 五、关键设计说明

1. **用户-陪诊师关联**: 陪诊师通过 `user_id` 关联用户表，同时在 `sys_user` 表中通过 `user_type` 区分用户类型

2. **订单状态机**: 使用 `status` 字段记录完整的订单生命周期

3. **抢单模式支持**: 订单创建时 `companion_id` 可为空，陪诊师在"抢单大厅"接单后更新

4. **资金流水**: 通过 `recharge_record` 和 `transaction_record` 完整记录用户资金变化

5. **陪诊师收入**: `companion_income` 表记录陪诊师每单收入，支持结算状态跟踪

6. **陪诊师标签**: 使用 JSON 数组存储在 `companion_profile.tags` 字段中

7. **陪诊师服务类别**: 使用 JSON 数组存储在 `companion_profile.services` 字段中
